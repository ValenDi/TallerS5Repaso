Introducción a Desarrollo Web

El desarrollo web es una de las ramas de la programación con mayor crecimiento, debido a su accesibilidad desde cualquier dispositivo y ubicación sin necesidad de instalaciones previas. El desarrollo web se trata de implementar sistemas, productos o aplicaciones usando tecnologías web.

Desarrollo Web vs. Diseño Web

  • Diseño Web: Determina la apariencia y la percepción de un sitio web. Se enfoca en la experiencia del usuario en un sitio web. Incluye diseño gráfico, navegación, y la gama de colores. Un diseñador web busca que el sitio sea fácil de usar y estéticamente agradable.
  • Desarrollo Web: Se encarga de las funciones y características del sitio web. Involucra la programación de la parte trasera del sitio, como sistemas de gestión de contenido y aplicaciones de base de datos. Permite a los visitantes interactuar y realizar acciones en el sitio.

Arquitectura de Software

La arquitectura de software es la estructura y organización del sistema de software, similar a los planos de un edificio. Un tipo de arquitectura fundamental es La Arquitectura Cliente-Servidor, donde un cliente hace peticiones a un servidor, el cual responde. Es fundamental para la mayoría de las comunicaciones en red.

Cliente:

Es un programa ejecutable que participa activamente en el establecimiento de las conexiones. Envía una petición al servidor y se queda esperando por una respuesta. Su tiempo de vida es finito, una vez que son servidas sus solicitudes, termina el trabajo.

Servidor:

Es un programa que ofrece un servicio que se puede obtener en una red. Acepta la petición desde la red, realiza el servicio y devuelve el resultado al solicitante. El servidor comienza su ejecución antes de comenzar la interacción con el cliente. Su tiempo de vida o de interacción es “interminable”.

Roles y Etapas en el Desarrollo Web

Roles

Incluyen desarrolladores frontend, backend, fullstack, diseñadores UI, administradores de bases de datos, project managers y testers. Cada rol tiene responsabilidades específicas. Antiguamente, todas las tareas en el desarrollo web recaían en un único rol, el Webmaster.

Etapas

  • Planificación: Documentación de requerimientos, diagramado de entidades, relaciones y funcionalidades.
  • Diseño: Definición de flujos de usuario y diseño de la interfaz gráfica.
  • Despliegue: Colocación del producto en producción después del testeo.
  • Soporte: Corrección de errores, implementación de cambios y mantenimiento.
